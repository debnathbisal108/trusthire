# TrustHire AI

**AI-powered employee background verification platform.**

Autonomously verifies employment history, education credentials, and public records using AI voice calls, email automation, and local LLM inference — with full GDPR/DPDP compliance.

## Quick start

### Prerequisites
- Docker + Docker Compose v2
- Python 3 (for key generation only)
- A Google Cloud project with OAuth 2.0 credentials

### 1. Clone and configure

```bash
git clone https://github.com/your-org/trusthire-ai.git
cd trusthire-ai
```

### 2. Run setup

```bash
chmod +x scripts/setup.sh
./scripts/setup.sh
```

This will:
- Copy `.env.example` → `.env`
- Auto-generate `NEXTAUTH_SECRET` and `ENCRYPTION_KEY`
- Start all Docker services
- Pull the Llama 3.2 model (~5 GB, first run only)
- Run database migrations

### 3. Add required credentials

Edit `.env` and add:

```bash
# Google OAuth (create at https://console.cloud.google.com)
# Authorized redirect URI: http://localhost:3000/api/auth/callback/google
GOOGLE_CLIENT_ID=your-client-id
GOOGLE_CLIENT_SECRET=your-client-secret
```

Then restart:
```bash
docker compose restart web
```

### 4. Open the app

```
http://localhost:3000
```

Sign in with Google — a workspace is automatically created.

## Architecture

```
Browser → Nginx → Next.js 15 (frontend)
                → FastAPI (backend)
                    → Celery workers
                        → Ollama (Llama 3.2, local LLM)
                        → faster-whisper (STT)
                        → Piper TTS
                        → Twilio (voice calls only)
                        → Resend / SMTP (email)
                    → PostgreSQL (primary DB)
                    → Redis (queue + cache)
                    → Qdrant (vector search)
                    → MinIO (file storage)
```

## Key workflows

### Adding a candidate
1. Go to **Candidates → Add candidate**
2. Fill in name, upload resume (PDF/DOCX/image)
3. AI automatically parses employment + education data
4. Grant consent on the candidate detail page
5. Click **Start verification**

### Verification pipeline
1. **Consent check** — all required consents verified
2. **Fraud analysis** — timeline overlaps, AI-generated resume detection
3. **Employment emails** — AI-generated emails to HR contacts
4. **Risk scoring** — weighted 0–100 score with explainable AI reasoning
5. **Report generation** — downloadable PDF

### Voice calls (requires Twilio)
Add to `.env`:
```bash
TWILIO_ACCOUNT_SID=AC...
TWILIO_AUTH_TOKEN=...
TWILIO_PHONE_NUMBER=+1...
WEBHOOK_BASE_URL=https://your-domain.com  # must be public
```

For local testing use ngrok:
```bash
ngrok http 8000
# Update WEBHOOK_BASE_URL with the ngrok HTTPS URL
```

## LLM configuration

Default: **Ollama + Llama 3.2 8B** (free, local)

```bash
# Switch to OpenAI (set in .env):
LLM_PROVIDER=openai
OPENAI_API_KEY=sk-...
```

Switch models without code changes:
```bash
LLM_MODEL=mistral:7b        # Lighter, faster
LLM_MODEL=qwen2.5:3b        # For 4 GB RAM machines
LLM_MODEL=llama3.2:8b       # Default (recommended)
```

## Deployment (production)

### Hetzner VPS (~€9/month)

```bash
# On VPS
apt install docker.io docker-compose-plugin
git clone https://github.com/your-org/trusthire-ai.git /opt/trusthire
cd /opt/trusthire
./scripts/setup.sh

# Set up SSL
apt install certbot python3-certbot-nginx
certbot --nginx -d yourdomain.com

# Set ENVIRONMENT=production in .env
docker compose up -d
```

### CI/CD

Push to `develop` → deploys to staging
Push to `main` → deploys to production

Configure GitHub secrets:
- `STAGING_HOST`, `STAGING_USER`, `STAGING_SSH_KEY`
- `PROD_HOST`, `PROD_USER`, `PROD_SSH_KEY`

## Legal compliance

TrustHire AI is designed to comply with:
- **GDPR** (EU) — explicit consent, right to erasure, audit logs
- **DPDP Act 2023** (India) — data principal rights
- **FCRA** (US) — if operating in the US, additional registration as a Consumer Reporting Agency may be required

**Important**: TrustHire AI never makes automated adverse employment decisions. All fraud flags and public record matches require human review. The system explicitly states "potential match requiring manual review" for all public record findings.
